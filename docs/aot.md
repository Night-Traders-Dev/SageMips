# SageMips AOT (Ahead-Of-Time) Optimizer

## Overview

The AOT optimizer applies static peephole optimizations to MIPS32 binaries before execution. It processes the entire binary in multiple passes, producing an optimized version that runs identically but with fewer instructions.

Both the C backend (`src/c/mips_aot.c`) and Sage backend (`src/sage/mips_aot.sage`) implement the same optimization passes.

## Optimization Passes

### Pass 1: NOP Elimination

Removes redundant `NOP` instructions (`sll $0, $0, 0`). NOPs are common in MIPS code as:
- Branch delay slot fillers
- Alignment padding
- Compiler-generated no-ops

**Before:**
```asm
    addiu $v0, $zero, 42
    nop
    nop
    syscall
```

**After:**
```asm
    addiu $v0, $zero, 42
    syscall
```

**Safety:** NOPs in branch delay slots are preserved (they're required for correct execution).

### Pass 2: Branch-to-Jump Conversion

Converts unconditional branches (`beq $0, $0, target`) into direct jumps (`j target`). The `beq $0, $0` idiom is generated by our assembler's `b` pseudo-instruction and by compilers for unconditional branching.

**Before:**
```asm
    beq   $zero, $zero, done   # always taken
    nop                         # delay slot
```

**After:**
```asm
    j     done
    nop                         # delay slot preserved
```

**Benefit:** J-type instructions are simpler to decode and dispatch. Eliminates register comparisons that always resolve to true.

### Pass 3: Constant Folding

Combines adjacent constant-loading instructions into single instructions:

**Pattern A — Consecutive ADDIU:**
```asm
    addiu $t0, $zero, 10     # $t0 = 10
    addiu $t1, $t0, 20       # $t1 = $t0 + 20 = 30
```
→
```asm
    addiu $t1, $zero, 30     # $t1 = 30
```

**Pattern B — ADDIU + ORI (used by li pseudo-instruction):**
```asm
    addiu $t0, $zero, 0x1234  # lower 16 bits
    ori   $t0, $t0, 0x5678     # upper 16 bits
```
→
```asm
    addiu $t0, $zero, 0x5678  # combined (when upper fits)
```

### Pass 4: Dead Code Elimination

Removes instructions after unconditional terminators (`j`, `jr`, `syscall`). Code after `syscall` or `jr $ra` can never be reached through normal control flow.

**Before:**
```asm
    syscall
    addiu $v0, $zero, 99     # dead — never executed
    jr $ra
```

**After:**
```asm
    syscall
```

## Usage

```bash
# Run with AOT optimization
sagemips run program.mips --aot

# AOT + JIT combined
sagemips run program.mips --aot --jit
```

AOT statistics are printed before execution:
```
AOT: 84 bytes optimized (3 NOPs, 2 consts, 1 branches, 4 dead)
```

## C Backend Implementation

**File:** `src/c/mips_aot.c`

- `mips_aot_init()` — Initializes the optimizer state
- `mips_aot_optimize()` — Runs all four passes on input code, returns optimized code length
- Individual pass functions: `pass_nop_elim()`, `pass_branch_opt()`, `pass_const_fold()`, `pass_dead_code()`

The optimizer works on a copy of the original code buffer. Each pass reads from the previous pass's output. This allows passes to be composed and tested independently.

## Sage Backend Implementation

**File:** `src/sage/mips_aot.sage`

The Sage version uses a class-based design with the same pass structure:
- `MipsAOT.optimize(code)` — Main entry point, returns optimized byte array
- `_pass_nop_elim()`, `_pass_const_fold()`, `_pass_branch_opt()` — Individual passes
- Helper methods: `_is_nop()`, `_is_branch()`, `_is_terminator()`

## Performance Impact

AOT optimization provides the largest benefit for:
- **Compiler-generated code**: Sage-generated MIPS assembly contains many NOPs in delay slots and constant-loading sequences
- **Large programs**: More opportunities for constant folding and dead code elimination
- **Unrolled loops**: Adjacent ADDIU instructions can be folded

Expected instruction reduction: **5-15%** for typical programs, **20-30%** for compiler-generated code.

## Safety Guarantees

All AOT passes preserve program semantics:
1. Branch delay slots are never removed
2. Constant folding only merges instructions where the intermediate register is not used elsewhere
3. Dead code elimination only removes code after unconditional terminators
4. All passes work on a copy — the original binary is never modified

## Build

The AOT optimizer is always compiled in. Enable at runtime with `--aot`:
```bash
make AOT=1          # ensure mips_aot.c is compiled
make                # standard build (AOT available at runtime)
```
